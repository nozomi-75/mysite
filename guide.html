Getting Familiar with Debian Releases

Stable

This release is suggested for normal use. Choose Debian Stable if you want your computer to just work for a prolonged period of time, without the risk of new packages breaking your workflow. In most cases, when people talk about Debian, they are referring to this.

Testing

Choose Debian Testing if you want to have recent software you use, and you are fine with constantly installing updates. Please be warned that they may be buggy at times.

Unstable

Debian Unstable would be handy if it is a must for you to have the absolute latest packages available. If you want to install Unstable, download the image for Stable and upgrade through Testing to Unstable by editing /etc/apt/sources.list. If timely security updates are important to you, don't use this.

Choosing your Installation Media

Network Install Image

This netinst ISO contains only a near-minimal Debian system. It is around a several hundred MB, and you can download as many packages you request during the installation process. It is suggested to use a wired connection in the case that your wireless network card won't be recognized.

CD/DVD Image

These image is the suggested approach, whereas only the first CD/DVD is required for the installation procedure. The additional images are optional and include extra packages that can be installed later.

Debian Live Images

They can start a real Debian system directly from the image without installation, so you can give Debian a try.

Creating a Bootable Media

After downloading the ISO file, you need to create a bootable media that you can use to install Debian. To create a bootable USB drive from Windows or MacOS, it is suggested to use Rufus and select DD writing mode. For Linux, you can simply run cp debian.iso /dev/sdX. This procedure will wipe the entire USB drive!

To burn the ISO to a disc in Windows, use IMGBurn or the built-in "Burn to disc" option in Windows 10. For MacOS, use the built-in Disc Utilityâ€”while for Linux, you can use xfburn or Brasero.

Booting from Installation Media

If you plan to dual-boot with another OS, both of them need to be installed with the same boot mode. You also need to arrange your boot order in the BIOS to prioritize your installation media. You should be able to boot to BIOS in the first few seconds of your computer starting up, by pressing either the F2, F8, F12, and Del keys.

Debian supports Secure Boot since Debian 11 Buster. If problems arise during Debian installation or boot process, turn off Secure boot before anything else. This guide suggests turning Secure Boot off.

Installation

Debian's Software Selection screen in the installer automatically checks Debian desktop environment. The desktop environment will be your system's user interface, from where you would be able to interact with it 'normally.' If you did not check other boxes, you will install the GNOME desktop. For this case, please check the screenshots of the included desktop environments below.

Networking

If your Wi-Fi does not work during the installation, you might need other proprietary firmware for your network card to function. Debian includes non-free firmware since Buster, so this should be the least of your problems. In any case that it won't work, please install Debian using a wired connection and setup wireless connectivity after installing.

Root and Sudo

The installation procedure will ask you later to set-up a root account. A root account is the superuser account of a computer or network, usually holding full control over all resources, files, and settings. It is the highest possible level of access that empowers a user to execute everything, edit every file, or configure any setting if need be.

Normally however, it is not necessary and you can simply skip the process. Doing so would give your account sudo (superuser do) permissions to perform certain operations that require root. Please be reminded however that you can potentially harm your system, in the case that you execute a harmful command or action.

If in any case that you set up a root account during install, simply run su -l to login to your root account. From that point, run adduser USERNAME sudo to add your account to the sudo group, enabling you to have the similar priveleges as above. Run exit to log-out of the root, log out of the desktop environment and log-in again.

Graphics Drivers

You can identify your graphics device by simply referring to the listed VGA compatible controller in lspci, or directly running lspci -nn | egrep -i "3d|display|vga". If you are using integrated graphics or an AMD dedicated GPU, setting up your drivers should be the least of your worries as it is already included.

If you happen to be using an NVIDIA graphics card, enable non-free in the /etc/apt/sources.list. Before installing the drivers, you must obtain the proper kernel headers for the driver to build with (linux-headers-amd64 or linux-headers-686). Install and run the nvidia-detect script from the non-free section of the repository to identify the recommended driver package to install. For NVIDIA Optimus graphics, please refer to this separate guide here.
